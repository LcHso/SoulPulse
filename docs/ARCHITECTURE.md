## 核心模块详解

### 1. 聊天服务 (ChatService)

聊天服务是系统的核心编排层，负责协调整个聊天流程。

```
用户消息 → ChatService.handle_user_message()
    │
    ├── 1. 获取 AI 角色信息
    │
    ├── 2. 获取/创建互动关系
    │
    ├── 3. 获取情绪状态
    │
    ├── 4. 构建上下文窗口
    │   ├── 获取对话摘要
    │   └── 获取最近消息
    │
    ├── 5. 持久化用户消息
    │
    ├── 6. 计算消息向量
    │
    ├── 7. 检索相关记忆
    │
    ├── 8. 检测关系锚点
    │
    ├── 9. 生成情绪指令
    │
    ├── 10. 调用 AI 生成回复
    │
    ├── 11. 持久化 AI 回复
    │
    ├── 12. 更新亲密度
    │
    ├── 13. 应用情绪变化
    │
    ├── 14. 提交事务
    │
    └── 15. 触发后台任务
        ├── 提取记忆
        ├── 提取锚点
        └── 生成摘要
```

### 2. 情绪引擎 (EmotionEngine)

情绪引擎管理用户与 AI 角色之间的情绪状态。

```
                    ┌─────────────────┐

                    │  愉悦度 → 0     │
                    │  激活度 → 0     │
                    │  思念 +0.03/h   │
                    │  安全感 -0.003/h│
                    └────────┬────────┘
                             │
                             ▼
┌──────────────────────────────────────────────────────┐
│                    情绪状态                           │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐              │

│  │ 0-100   │  │ -1~1    │  │ -1~1    │              │
│  └────┬────┘  └────┬────┘  └────┬────┘              │
│       │            │            │                   │
```

### 3. 记忆系统 (MemoryService)

记忆系统实现长期记忆的存储和检索。

#### 记忆流程

```
用户对话 → 提取记忆 → 向量化 → 存储
    │                              │
    │                              ├── SQLite (元数据)
    │                              └── ChromaDB (向量)
    │
    ▼
下次对话时
    │
    ├── 1. 向量化用户消息
    │
    ├── 2. ChromaDB 语义检索
    │
    ├── 3. 亲密度过滤
    │   ├── Lv < 6: 仅事实记忆
    │   └── Lv ≥ 6: 事实 + 情感
    │
    ├── 4. 计算记忆年龄
    │
    └── 5. 格式化注入提示词
        ├── fresh: 精确回忆
        ├── fading: 模糊回忆
        └── distant: 核心感受
```

## 性能优化

### 数据库优化

1. **索引策略**
   - 所有外键字段建立索引
   - 复合索引用于常见查询模式
   - 使用 WAL 模式提高并发性能

2. **查询优化**
   - 使用异步 SQLAlchemy
   - 批量查询减少数据库往返
   - 游标分页避免 OFFSET

### 缓存策略

1. **API 缓存**
   - 帖子列表缓存（5 分钟）
   - AI 角色信息缓存

2. **向量缓存**
   - 预计算消息向量
   - 复用向量进行记忆检索

### 后台任务

1. **异步处理**
   - 记忆提取（fire-and-forget）
   - 摘要生成（fire-and-forget）
   - 图片生成（异步轮询）

2. **调度优化**
   - 错开任务执行时间
   - 批量处理减少 API 调用

## 安全设计

### 认证与授权

1. **JWT 认证**
   - HS256 算法
   - 24 小时过期时间
   - 自动刷新机制

2. **权限控制**
   - 用户隔离（数据按 user_id 过滤）
   - 管理员权限（is_admin 标识）

### 数据安全

1. **密码存储**
   - bcrypt 哈希
   - 自动加盐

2. **API 密钥管理**
   - 环境变量存储
   - 不写入代码库

## 扩展性设计

### 水平扩展

1. **无状态服务**
   - FastAPI 服务无状态
   - 会话信息存储在 JWT

2. **数据库分离**
   - 可迁移到 PostgreSQL/MySQL
   - ChromaDB 可替换为其他向量库

### 功能扩展

1. **新增 AI 角色**
   - 数据库添加记录
   - 配置性格和视觉标签

2. **新增触发类型**
   - 扩展 check_proactive_triggers
   - 添加对应的执行器函数

3. **新增情绪维度**
   - 扩展 EmotionState 模型
   - 更新情绪计算逻辑
